# CR460 ## Cours 2 et 3
## Mode PaaS - Déploiement de code applicatif
## Démo - Github Outil de gestion de code source
## Démo - Github - Repo * Un repo est un système de fichiers. * Un commit crée une version à laquelle on peut revenir dans le futur, permet les rollback et la création de versions différentes, doit spécifier le contexte du commit.
## Démo - Github - Repo * Un repo est un système de fichiers. * Cloner un repo localement permet de travailler le code et tester avant de le commit pour publier (Commit and push, commit est juste local et push envoie au serveur). * Donc on a un endroit pour pousser des fichiers, les modifier et garder une historique. Le processus de commit permet au repo de savoir le moment dans le temps ou le c ode a été modifié.
## Démo - Github - Branch * Le concept de branche permet d’avoir plusieurs versions du code existant en parallèle. * Avant de pousser le code en production, il est bien de passer par un environnement de staging / assurance qualité afin de ne pas briser l'environnement de production.
## Démo - Github - Pull Request / Fork * Le pull request est une demande visant à amener le contenu d’une branche plus basse vers le haut. * Le processus comprend des vérifications, notamment choisir quelles versions de fichiers à conserver; ceci « efface » la branche enfant. * Un fork permet de prendre un repo et d’en faire une copie dans notre repo personnel, le modifier et ensuite faire un pull request au repo principal. * Attention aux droits de modifications selon la branche.
## Démo - Heroku Fournisseur Paas Permet de rouler du code sans se soucier de l’environnement / infrastructure.

## Démo - Heroku * Les applications sont les contenants où on pousse le code pour ensuite l’exécuter. * Premièrement il faut créer une coquille vide, ensuite on établit un lien entre Heroku et le repo (pipeline).
## Démo - Heroku * Procfile sert à définir la commande à rouler, sert pour qu'Heroku sache quoi faire avec le code dans le repo (process file). * Dans Heroku, il faut choisir une méthode de déploiement. On peut lier différents comptes et même configurer un déploiement automatique lorsque le code est modifié. * Heroku peut être dirigé afin de prendre du code diffèrent à un moment diffèrent et le rouler sur un url diffèrent.
